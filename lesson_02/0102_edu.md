# S01E02 FindHim — educational walkthrough (agent + tools)

This note explains how the final `findhim` solution works, with focus on **agent programming**, **tool design**, and a short frontend section.

---

## 0) One-page cheat sheet (quick rebuild)

### What to build
- A Django service with a **supervisor LLM** that uses **3 tools**:
  1. `identify_closest_suspect()`
  2. `get_access_level(name, surname, birth_year)`
  3. `submit_answer(name, surname, access_level, power_plant)`

### Minimal flow
1. Load suspects from `0102task_context/suspect_list.json`.
2. Load power plants from `0102task_context/powerplants.json`.
3. Tool #1 loops all suspects:
   - call `/api/location`,
   - parse coordinates (`lat/lon` **or** `latitude/longitude`),
   - compute Haversine to all plants,
   - return best candidate as JSON.
4. Tool #2 calls `/api/accesslevel` with `birthYear`.
5. Tool #3 writes `answer.json`, then POSTs `/verify`.
6. Return final output + tool trace + API log to frontend.

### Core constants
```python
FINDHIM_MODEL_OVERRIDE = ""
DEFAULT_FINDHIM_MODEL = "openai/gpt-5.4-mini"
MAX_AGENT_ITERATIONS = 12
```

### Must-have prompt constraints
- “Use tools only.”
- “Execute exactly identify -> access -> submit.”
- “Do not modify name/surname/birth year/power plant code.”
- “Return short final status after submit.”

### Manual loop skeleton
```python
for i in range(MAX_AGENT_ITERATIONS):
```

```python
api_log.append({
    "endpoint": endpoint,

```

### HTMX wiring


### Common failure checklist
- `AIDEVSKEY` missing.
- `/api/location` returned unsupported shape and no coords parsed.
- Wrong payload keys in `/verify` (`accessLevel`, `powerPlant` exact casing).
- Agent loops too long (increase prompt strictness, not only iteration limit).

## 1) Problem we solved

Goal of the quest:
- find which suspect was seen closest to a nuclear power plant,
- get their `accessLevel`,
- submit:

```json
{
  "task": "findhim",
  "answer": {
    "name": "...",
    "surname": "...",
    "accessLevel": 1,
    "powerPlant": "PWR....PL"
  }
}
```

Main challenge was not just HTTP calls, but **reliable orchestration**:
- multiple suspects,
- multiple locations per suspect,
- multiple plants,
- and one final exact payload.

---

## 2) Final architecture

The final service is a **supervisor-driven tool-calling agent**:

1. LLM supervisor decides which tool to call.
2. Tools do deterministic work (HTTP + calculations).
3. Manual loop executes tool calls and feeds outputs back.
4. Agent ends with short final status/flag.

This gives us both:
- course goal: agentic workflow,
- production safety: deterministic tools for critical logic.

Resolver:

```python
def _resolve_model(model: str | None = None) -> str:
    return (FINDHIM_MODEL_OVERRIDE or model or DEFAULT_FINDHIM_MODEL).strip()
```

Interpretation:
- set `FINDHIM_MODEL_OVERRIDE = "openai/gpt-5-mini"` to force a model for this module,
- leave it empty to use default `openai/gpt-5.4-mini`.

## 4) Tool design — key principle

Tools are where “real work” happens.  
LLM chooses tools, but tools execute strict logic.

### Tool A: `identify_closest_suspect`

Responsibilities:
- call `/api/location` for each suspect,
- parse coordinates (supports both schemas),
- compute Haversine distance to all plants,
- return best candidate JSON.

Core pattern:

```python
@tool
def identify_closest_suspect() -> str:
    best = {"distanceKm": float("inf")}
    for suspect in suspects:
        status, data, body = _post_json(endpoint="/api/location", payload=payload, ...)
        coords = _extract_coords(data)
        for lat, lon in coords:
            for city, plant in plants.items():
                distance_km = _haversine(lat, lon, plant["lat"], plant["lon"])
                if distance_km < best["distanceKm"]:
                    best = {...}
    return json.dumps(best, ensure_ascii=False)
```

### Tool B: `get_access_level`

Responsibilities:
- call `/api/accesslevel`,
- validate expected JSON,
- return JSON text for agent context.

```python
@tool
def get_access_level(name: str, surname: str, birth_year: int) -> str:
    payload = {"apikey": _api_key(), "name": name, "surname": surname, "birthYear": int(birth_year)}
    status, data, body = _post_json(endpoint="/api/accesslevel", payload=payload, ...)
    if status >= 400:
        raise RuntimeError(...)
    return json.dumps(data, ensure_ascii=False)
```

### Tool C: `submit_answer`

Responsibilities:
- create final payload,
- save payload to `answer.json` before submit,
- call `/verify`,
- return server response text.

```python
@tool
def submit_answer(name: str, surname: str, access_level: int, power_plant: str) -> str:
    payload = {"apikey": ..., "task": "findhim", "answer": {...}}
    answer_path.write_text(json.dumps(payload, ensure_ascii=False, indent=2), encoding="utf-8")
    status, _, body = _post_json(endpoint="/verify", payload=payload, ...)
    return body
```

## 5) Why `_extract_coords` was critical

A major reliability fix:

```python
def _extract_coords(location_payload: Any) -> list[tuple[float, float]]:
    lat = item.get("lat", item.get("latitude"))
    lon = item.get("lon", item.get("longitude"))
```

Without this, agent can parse zero points if API key names differ, and the run fails with “no suspect”.

## 6) Supervisor prompt strategy

Prompt gives strict task contract and ordering:

```python
supervisor_prompt = (
    "You are the FindHim supervisor agent.\n"
    "You must execute exactly this sequence:\n"
    "1) identify_closest_suspect\n"
    "2) get_access_level\n"
    "3) submit_answer\n"
    "Rules: do not modify identity, birth year, or power plant code."
)
```

Why this works:
- narrows agent behavior,
- keeps freedom only where needed (tool arguments),
- reduces hallucinated branches and retries.

## 12) Key conceptual lessons

- Agentic != random autonomy; best results come from **constrained tool contracts**.
- Deterministic logic should live in tools; LLM should orchestrate.
- Most failures are data-contract failures (field names, payload shape), not “AI quality”.
- Observable systems (logs + traces) are dramatically easier to fix.
